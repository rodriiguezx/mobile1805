1. Quais problemas foram identificados no código original?
Muitas responsabilidades em uma única classe, Uso de strings fixas, Código pouco escalável, Excesso de condicionais.
2. Como a refatoração melhorou a organização e legibilidade?
Separou responsabilidades, Deixou o código mais modular, Reduziu complexidade, Melhorou a clareza do fluxo.
3. Quais padrões de design foram aplicados e por quê?
Strategy Pattern para separar comportamentos de cada tipo de pedido, Princípios SOLID, principalmente SRP e Open/Closed.
4. Como essas mudanças impactam manutenção e escalabilidade?
Facilita manutenção, Permite adicionar novos status facilmente, Reduz riscos de bugs, Melhora escalabilidade do sistema.
5. Quais boas práticas foram aplicadas?
Separação de responsabilidades, Uso de enums, Modularização, Baixo acoplamento, Código mais legível.
